Explanation of the flow

Start: The user gives the AI a goal (not a single prompt). 
Discover: The agent determines what needs to be done to reach the goal. 
Plan: It decides the steps and order. 
Execute: It performs the actions (writes code, generates text, etc.). 
Verify: A strict gate (test, rubric, measurement) checks the output. 
Yes → goal achieved, stop and report. 
No → log state, then iterate by fixing the weakest point.


Iterate: The result is fed back into the cycle – this repeats until the goal is met or a hard stop is reached (max 8 tries, token cap, etc.).

Key components (coloured in the diagram)

Automation – the scheduler / trigger (e.g., cron, GitHub Actions, /loop) 
Skill – reusable instruction file (rules, patterns, constraints) 
Sub‑agents – separate writer (fast/cheap) and reviewer (strict/thorough) 
Connectors – real‑world actions (open a PR, create a ticket, send a message)
