Engineering

How the Agentic Loop works

7-Phase Autonomous Loop

DISCOVER → baseline verification (tests, lint, types)
PLAN → AI plans the highest-impact fix
ACT → one targeted change per iteration
VERIFY → run gate commands, check exit codes
REVISE → compare results, detect progress
REVIEW → reviewer sub-agent audits the diff
HALT → stop on budget, stall, or user abort

Dual-Mode Autonomy

ASK Mode — every tool action requires your explicit permission. Full visibility, full control.

AUTO Mode — AI iterates autonomously with safety gates. Read tools always allowed; write/exec pass through the autonomy manager. Dangerous commands (rm -rf, git push --force, DROP TABLE) are hard-blocked regardless of mode.

Open Settings → Models & Providers in the IDE. Enter your API key for each provider. Keys are stored in Windows Credential Manager, encrypted with DPAPI. They never leave your machine.

In the Windows Credential Manager, encrypted at rest by the operating system with DPAPI and tied to your Windows account. Keys are never written to a plaintext config file, never logged, and never exported. The IDE also keeps an AES-256-GCM encrypted backup at ~/.cortex/keys.enc.
